title: "Quickstart: generate your first image in minutes"
description: "Learn how to set up your OpenAI API key, configure your first session, and generate your first image with GPT Image Playground in minutes."
---

This guide walks you through everything you need to go from opening GPT Image Playground to generating your first image. By the end, you will have your API key configured, your generation parameters set, and a completed image task in your history.

<Steps>
  <Step title="Open the app">
    Visit the live deployment at [gpt-image-playground.cooksleep.dev](https://gpt-image-playground.cooksleep.dev). No sign-up or installation is required ‚Äî the app runs entirely in your browser.

    If you prefer to run your own instance, you can deploy to Vercel in one click or run it with Docker. See the [Vercel deployment guide](/deploy/vercel) for details.
  </Step>

  <Step title="Enter your API key">
    Click the gear icon in the top-right corner of the app to open **Settings**. Paste your OpenAI API key into the **API Key** field and click outside the panel to save.

    <Note>
      Your API key is stored only in your browser's `localStorage`. It is never sent to any server operated by GPT Image Playground ‚Äî it goes directly from your browser to the OpenAI API when you generate an image. GPT Image Playground has no server-side component.
    </Note>
  </Step>

  <Step title="Choose your API mode">
    In **Settings**, select the API mode that matches the model you want to use:

    - **Images API** ‚Äî Calls `/v1/images/generations` and `/v1/images/edits`. Uses the `gpt-image-2` model by default. This is the default mode and the best starting point for most users.
    - **Responses API** ‚Äî Calls `/v1/responses` and uses the `image_generation` tool. Uses the `gpt-5.5` model by default. Choose this mode if you want to use a text model that supports the `image_generation` tool for multimodal workflows.

    You can switch modes at any time from Settings without losing your history.
  </Step>

  <Step title="Write a prompt">
    The input bar runs along the bottom of the screen. Click the text area and type a description of the image you want to generate. Be as specific as you like ‚Äî prompt quality directly affects output quality.

    You can also attach reference images for editing by clicking the image icon to the left of the input bar, or by pasting or dragging images directly onto the page.
  </Step>

  <Step title="Set generation parameters">
    Click the parameter controls above the input bar to adjust:

    - **Size** ‚Äî Choose `auto` to let the model decide, or select a resolution based on `1K`, `2K`, or `4K` quality tiers combined with common aspect ratios. You can also enter custom width and height values.
    - **Quality** ‚Äî `auto`, `low`, `medium`, or `high`. Higher quality increases generation time and cost.
    - **Output format** ‚Äî `PNG`, `JPEG`, or `WebP`.
    - **Batch count (n)** ‚Äî The number of images to generate in a single request.

    All parameter values persist between sessions so you do not need to reconfigure them each time.
  </Step>

  <Step title="Generate your image">
    Click **Generate**. A new task card appears at the top of the history grid immediately, showing a running indicator while the request is in progress. When the API responds, the card updates with thumbnails of the generated images, the prompt, parameters, and elapsed time.

    Click any task card to open the detail view, where you can inspect full-size images, download them, reuse the configuration, or add an output image to your reference images for a follow-up edit.
  </Step>
</Steps>

<Tip>
  You can pre-fill your API key, API URL, and mode using URL query parameters ‚Äî useful for bookmarks or sharing a pre-configured link:

  ```text
  https://gpt-image-playground.cooksleep.dev?apiUrl=https://api.openai.com/v1&apiKey=sk-...&apiMode=images
  ```

  Supported parameters: `apiUrl`, `apiKey`, `apiMode` (`images` or `responses`), and `codexCli` (`true` or `false`).
</Tip>
